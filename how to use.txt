SECTION 4: USING THE ADMIN DASHBOARD
================================================================================

TAB 1: MENU MANAGEMENT
-----------------------
This is where you control what food items customers can order.

TO ADD A NEW MENU ITEM:
1. Scroll to "Add New Item" section (bottom of page)
2. Fill in:
   - Item Name: e.g., "Jollof Rice Special"
   - Price (‚Ç¶): e.g., 5000 (enter numbers only, no commas)
   - Description: Brief description of the dish
3. Click "Add Item"
4. The item appears in the menu list above

TO UPLOAD A MENU PHOTO:
1. Go to "Menu Image" section
2. Click "Choose File"
3. Select a nice photo of your menu or food
4. Click "Upload Menu Image"
5. The photo will be shown to customers

TO MARK AN ITEM UNAVAILABLE (Sold Out):
1. Find the item in the menu list
2. Click the red "Mark Unavailable" button
3. Customers won't see this item until you mark it available again

TO MARK AN ITEM AVAILABLE AGAIN:
1. Find the item in the menu list
2. Click the green "Mark Available" button
3. Customers can now order it

TO DELETE AN ITEM PERMANENTLY:
1. Find the item in the menu list
2. Click the "Delete" button (‚ö†Ô∏è This cannot be undone!)

TAB 2: ORDERS
-------------
This shows all customer orders in real-time.

UNDERSTANDING ORDER STATUS:
- ‚è≥ Processing: Order just came in, waiting for payment verification
- üç¥ Prepared: Kitchen has finished preparing the food
- üöö Out for Delivery: Order is with the rider
- ‚úÖ Delivered: Customer received the order
- ‚ùå Cancelled: Order was cancelled

TO UPDATE ORDER STATUS:
1. Find the order in the list
2. Click "Update Status" dropdown
3. Select new status:
   - "Prepared" when kitchen is done
   - "Out for Delivery" when rider picks up
   - "Delivered" when customer confirms receipt
4. If selecting "Out for Delivery", enter rider's phone number
5. Click "Update"
6. Customer automatically gets a Telegram notification!

TO CANCEL AN ORDER:
1. Find the order
2. Click "Cancel Order" button
3. Customer gets notified automatically

TO MESSAGE A CUSTOMER:
1. Find their order
2. Click "Send Query" button
3. Type your message (e.g., "Is this address correct?")
4. Click "Send"
5. Message goes directly to their Telegram

TAB 3: PAYMENTS
---------------
This is where you verify customer payment receipts.

PAYMENT VERIFICATION WORKFLOW:
1. Customer places order on Telegram
2. Customer pays to your bank account
3. Customer uploads payment receipt photo to the bot
4. Receipt appears here in "Pending Verification" section

TO APPROVE A PAYMENT:
1. Look at the receipt image
2. Check if:
   - Amount matches the order total
   - Payment went to your account
   - Receipt looks genuine
3. If everything is correct, click "‚úÖ Approve"
4. Customer gets instant confirmation on Telegram
5. Order status changes to "Processing"

TO REJECT A PAYMENT:
1. If receipt is fake, wrong amount, or unclear
2. Click "‚ùå Deny"
3. Order stays as "Pending Payment"
4. Contact customer via "Send Query" to clarify

SECTION 7: TROUBLESHOOTING COMMON ISSUES
================================================================================

PROBLEM: Bot doesn't respond on Telegram
SOLUTION:
- Check if Terminal shows "ü§ñ Telegram bot starting..."
- Verify BOT_TOKEN in .env file is correct
- Make sure you started the bot with: python bot.py
- Try stopping (Ctrl+C) and restarting the bot

PROBLEM: Admin dashboard is empty/blank
SOLUTION:
- Make sure bot is running (see Terminal window)
- Check that you're opening index.html (double-click it)
- Refresh the browser page (press F5 or Cmd+R)
- Make sure PORT in .env is 3000

PROBLEM: Orders not showing up
SOLUTION:
- Click "Refresh Orders" button in dashboard
- Make sure customer actually completed checkout
- Check Terminal window for any error messages

PROBLEM: Customer can't see menu photos
SOLUTION:
- Make sure photos are in the "uploads" folder
- Photos must match the exact names listed in bot.py
- Try uploading menu image again in Menu Management tab

PROBLEM: "Port already in use" error
SOLUTION:
- Another program is using port 3000
- Change PORT in .env to 3001 or 3002
- Restart the bot

PROBLEM: Customer didn't get payment confirmation
SOLUTION:
- Check "Payments" tab - verify the receipt manually
- If you already clicked Approve, check "Orders" tab
- Customer can check status by sending /start to bot again
